Destination File Table

The Destination File Table is defined by DS_DestFileTable_t. This table contains a description and DS_DEST_FILE_CNT entries. Each entry is defined by DS_DestFileEntry_t and described below.

  1. Pathname - The path portion of the filename
  2. BaseName - The optional base portion of the filename
  3. Extension - The extension portion of the filename
  4. FileNameType - The filename type DS_BY_COUNT or DS_BY_TIME
  5. EnableState - The state of this entry DS_ENABLED or DS_DISABLED
  6. MaxFileSize - The maximum number of bytes to store in this file
  7. MaxFileAge - The number of seconds that this file remains open
  8. SequenceCount - The sequence count portion of the filename if the file type is DS_BY_COUNT



Filter Table

The Filter Table is defined by DS_FilterTable_t. This table contains a description and DS_PACKETS_IN_FILTER_TABLE entries. Each entry is defined by DS_PacketEntry_t. Each Packet entry contains the Message ID for DS to store and DS_FILTERS_PER_PACKET entries defined by DS_FilterParms_t. These entries are described below.

  1. FileTableIndex - The zero based index into the Destination File table
  2. FilterType - The type of filter used by this entry DS_BY_COUNT or DS_BY_TIME
  3. Algorithm_N - The N argument passed into the Filter algorithm
  4. Algorithm_X - The X argument passed into the Filter algorithm
  5. Algorithm_O - The O argument passed into the Filter algorithm



The Data Storage (DS) application attempts to load the File and Filter Tables from DS_DEF_DEST_FILENAME and DS_DEF_FILTER_FILENAME upon startup. If the load fails for either of these tables, DS will still start in the mode of operation as defined by the DS_DEF_ENABLE_STATE configuration parameter. If this configuration parameter is set to ENABLED mode and either the File and/or Filter Table failed to load, DS will not store any messages until both tables are loaded with valid data. This can be accomplished by using the cFE Table Services commands after creating a table load image file and uploading it to the spacecraft's file system. Until both tables are loaded with valid data, DS will increment the #DS_IGNOREDPKTCNT for each received packet that is discarded.
