ULR_RESERVED_EID 

Value of zero is reserved, and should not be used.

ULR_INIT_INF_EID 

'Initialized. Version %d.%d.%d.%d'

Event Message:
'Initialized. Version %d.%d.%d.%d'
Type: INFORMATION
Cause:

This event message is issued when the ULR task has completed initialization.

The first d field contains the application major version defined in #ULR_MAJOR_VERSION. The second d field contains the application minor version defined in #ULR_MINOR_VERSION. The third d field contains the application revision number defined in #ULR_REVISION. The fourth d field contains the application revision number defined in ULR_MISSION_REV.

ULR_CMD_NOOP_EID 

'Recvd NOOP. Version %d.%d.%d.%d'

Event Message:
'Recvd NOOP. Version %d.%d.%d.%d'
Type: INFORMATION
Cause:

This event message is issued when the CFS ULR Task receives a NoOp command.

The first u field contains the application major version defined in #ULR_MAJOR_VERSION. The first d field contains the application major version defined in #ULR_MAJOR_VERSION. The second d field contains the application minor version defined in #ULR_MINOR_VERSION. The third d field contains the application revision number defined in #ULR_REVISION. The fourth d field contains the application revision number defined in ULR_MISSION_REV.

ULR_SUBSCRIBE_ERR_EID 

'%s Pipe failed to subscribe to %s. (0x%08X)'

Event Message:
'%s Pipe failed to subscribe to %s. (0x%08X)'
Type: ERROR
Cause:

This event message is issued when the CFS ULR Task fails to subscribe to a message at initialization.

The first s field contains the name of the pipe. The second s field contains the macro name of the message. The 32 bit hexadecimal number is the error code returned by CFE.

ULR_PIPE_INIT_ERR_EID 

'Failed to create %s pipe (0x%08X)'

Event Message:
'Failed to create %s pipe (0x%08X'
Type: ERROR
Cause:

This event message is issued when the CFS ULR Task fails to create a CFE Software Bus pipe at initialization.

The s field contains the name of the pipe. The 32 bit hexadecimal number is the error code returned by CFE.

ULR_RCVMSG_ERR_EID 

'%s pipe read error (0x%08X).'

Event Message:
'%s pipe read error (0x%08X).'
Type: ERROR
Cause:

This event message is issued when the CFS ULR Task has had an error reading from a pipe.

The s field contains the name of the pipe. The 32 bit hexadecimal number is the error code returned by CFE.

ULR_MSGID_ERR_EID 

'Recvd invalid %s msgId (0x%04X)'

Event Message:
'Recvd invalid %s msgId (0x%04X)'
Type: ERROR
Cause:

This event message is issued when the CFS ULR Task has received an invalid message ID.

The s field contains the name of the pipe. The 16 bit hexadecimal number is the actual message ID received.

ULR_CC_ERR_EID 

'Recvd invalid command code (%u)'

Event Message:
'Recvd invalid command code (%u)'
Type: ERROR
Cause:

This event message is issued when the CFS ULR Task has received an invalid message ID.

The u field contains the actual command code received.

ULR_MSGLEN_ERR_EID 

'Rcvd invalid msgLen: msgId=0x%08X, cmdCode=%d, msgLen=%d, expectedLen=%d"

Event Message:
'Rcvd invalid msgLen: msgId=0x%08X, cmdCode=%d, msgLen=%d, expectedLen=%d"
Type: ERROR
Cause:

This event message is issued when the CFS ULRTask has received a message with an invalid length.

The first d field contains the message ID. The second d field contains the command code. The third d field contains the actual length. The fourth d field contains the expected length.

ULR_UNEXPECTED_DATA_BEFORE_VERSION_ID 

'Received 0x%02x before version ID.'

Event Message:
'Received 0x%02x before version ID.' 
\par Type: ERROR

\par Cause:
This event message is issued when the parser encounters data before
the version ID is read.  This usually indicates that some of the
message was either dropped or was injected with noise.  This should
not latch or be a fatal error since the parser will just resync on
the beginning of the next message.

'

Event Message:
'UartMessage.Checksum hhu != checksum hhu
' 
\par Type: ERROR

\par Cause:
This event message is issued when the parser calculates a checksum
that does not match the expected checksum reported by the device.
This should not latch or be a fatal error since the parser will just
resync on the beginning of the next message.
