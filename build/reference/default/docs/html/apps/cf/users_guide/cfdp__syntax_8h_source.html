53 /* Put Requests use this syntax:
54  * Put [-class1] <source_file> <destination_id> [dest_file]
55  * Examples:
56  * Put myfile 23 yourfile -- sends 'myfile' to CFDP node 23, and
57  * calls it 'yourfile'. (Service class 2)
58  * Put sun.data 23 -- sends 'sun.data' to CFDP node 23, and
59  * calls it 'sun.data'. (Service class 2)
60  * Put -class1 sun.data 23 -- same as previous, except Service Class 1
61  * (i.e. no retransmissions by CFDP)
62  * Put sun.data 101.158 -- if the partner's Entity-ID exceeds 1 byte,
63  * use dotted-decimal format.
64  * Put -class1 abc 23 def -- Uses service class 1 to transfer 'abc'
65  * to node 23 and call it 'def'.
66  */
67 
68 /* Suspend, Resume, Cancel, Abandon, and Report Requests all use the
69  * same syntax. (Only 'cancel' will be shown):
70  * Cancel <transaction>
71  * Examples:
72  * Cancel 23_808 -- cancels the transaction which CFDP node 23 started
73  * and assigned transaction-sequence-number = 808.
74  * (a transaction is uniquely identified by the
75  * combination of the Source-ID and trans-seq-number)
76  * Cancel 101.158_3 -- ID=101.158, trans-seq-number=3.
77  * Cancel all -- cancels all transactions.
78  */
