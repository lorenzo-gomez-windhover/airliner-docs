14  * BIG PICTURE: The intent is to allow most of the CFDP library to work
15  * with PDUs as if they were standard "C" data structures. Each incoming
16  * (raw) PDU is immediately converted to a C data structure, and each
17  * outgoing PDU is converted from the internal data structure to raw bytes
18  * at the last possible moment.
19  * Hopefully, this allows the library source code to be easier to read,
20  * and, perhaps, more efficient as well.
21  * SUMMARY: There are routines for converting from C data structures to
22  * raw PDU bytes (assembly), and vice versa (disassembly).
23  * Plus, there are a few special purpose routines.
24  * CHANGES:
25  * 2007_07_31 Tim Ray
26  * - Official release. (see pre_release_changes.log for earlier changes)
