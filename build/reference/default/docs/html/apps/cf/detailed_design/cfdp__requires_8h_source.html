124 /* This callback routine allows the library user to provide their own
125  * response to Indications (either in place of, or in addition to, the
126  * default library response).
127  * BACKGROUND: CFDP specifies a discrete set of Indications that are to
128  * be issued by the protocol to the User. For example, when a transaction
129  * completes, an Indication is issued. CFDP does not specify what actions to
130  * take in response to each Indication; that is implementation-dependent.
131  * NOTE: Independent of this callback routine, the library will output
132  * a simple text message in response to each Indication (unless the
133  * 'MSG_INDICATIONS' message-class has been disabled in 'cfdp_config.h').
134  */

151 /* The library uses callback routines to output messages. There are 4
152  * callback routines - one for each severity-level (debug, info, warning,
153  * and error). All of the message callbacks match the POSIX 'printf'
154  * routine. The library user may register a *separate* callback routine for
155  * each severity level. Or, register a *single* callback routine to handle
156  * all levels. Or, accept the default callback, which is 'printf'.
157  */
158
