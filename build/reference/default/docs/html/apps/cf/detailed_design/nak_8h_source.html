13  * BIGGER PICTURE: In the formal CFDP protocol, the Nak is a Protocol Data
14  * Unit indicating requests for retransmissions. Within this software
15  * library, the Nak structure has the same contents as a Nak PDU, but it
16  * is used in a broader way. The Sender uses the Nak structure to indicate
17  * what data needs to be sent (throughout the life of the transaction, not
18  * just when feedback is received from the Receiver). (The Receiver uses
19  * the Nak structure in the usual way).
20  * SUMMARY: This module maintains NAK-Pdu data (represented as a C structure);
21  * i.e. it keeps track of:
22  * 1) whether or not Metadata needs to be retransmitted
23  * 2) any gaps in the file-data (which File-data needs to be retrans).
24  * This module is designed so that separate transactions can maintain
25  * separate Nak-data (the Nak-data is passed in/out with each routine).
26  * NOTE: Only 'deferred' (comprehensive) Nak-mode is supported!
27  * USAGE:
28  * Call 'nak__init' once for each transaction before calling any of the
29  * other routines.
30  * When copying a Nak structure, use the copy_nak routine below (unless
31  * you enjoy looking through core dumps).
32  * CHANGES:
33  * 2007_07_31 Tim Ray
34  * - Official release. (see pre_release_changes.log for earlier changes)
35  */
