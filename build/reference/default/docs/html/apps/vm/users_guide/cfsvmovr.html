VM Application Overview

The VM Application (VM) application enables safe vehicle commanding, additionally detects failure and executes contingent action.

VM Design Overview

The VM Application (VM) has three components.

  1. System Routines
  2. Arming State Machine
  3. Navigation State Machine

System Routines

The Routines performed by VM include battery management, listening to RC commanding, checking for citicality and warnings.

Arming State Machine

The state machine initializes at a start state (Init) then moves to default state (Standby) and waits for further transitions.
The state machine handles trasitions between states and guards against illeagal transitions of the states.
The states that can be found in this state machine are:

  1. Init
  2. Standby
  3. Armed
  4. StandbyError
  5. ArmedError

Navigation State Machine

The state machine initializes at a start state (Init) then moves to default state (Manual) and waits for further transitions.
The transitions can be invoked by either RC or from ground station.
The state machine handles trasitions between states and guards against illeagal transitions of the states.
The states that can be found in this state machine are:

  1. Init
  2. Manual
  3. Altitude Control
  4. Position Control
  5. Acrobatic
  6. Rattitude
  7. Stabilize
  8. Auto Takeoff
  9. Auto Land
  10. Auto Loiter
  11. Auto RTL
