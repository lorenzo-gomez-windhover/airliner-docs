MPU9250_RESERVED_EID 

Value of zero is reserved, and should not be used.

MPU9250_INIT_INF_EID 

'Initialized. Version %d.%d.%d.%d'

Event Message:
'Initialized. Version %d.%d.%d.%d'
Type: INFORMATION
Cause:

This event message is issued when the MPU9250 task has completed initialization.

The first d field contains the application major version defined in #MPU9250_MAJOR_VERSION. The second d field contains the application minor version defined in #MPU9250_MINOR_VERSION. The third d field contains the application revision number defined in #MPU9250_REVISION. The fourth d field contains the application revision number defined in MPU9250_MISSION_REV.

MPU9250_CMD_NOOP_EID 

'Recvd NOOP. Version %d.%d.%d.%d'

Event Message:
'Recvd NOOP. Version %d.%d.%d.%d'
Type: INFORMATION
Cause:

This event message is issued when the CFS MPU9250 Task receives a NoOp command.

The first u field contains the application major version defined in #MPU9250_MAJOR_VERSION. The first d field contains the application major version defined in #MPU9250_MAJOR_VERSION. The second d field contains the application minor version defined in #MPU9250_MINOR_VERSION. The third d field contains the application revision number defined in #MPU9250_REVISION. The fourth d field contains the application revision number defined in MPU9250_MISSION_REV.

MPU9250_SUBSCRIBE_ERR_EID 

'%s Pipe failed to subscribe to %s. (0x%08X)'

Event Message:
'%s Pipe failed to subscribe to %s. (0x%08X)'
Type: ERROR
Cause:

This event message is issued when the CFS MPU9250 Task fails to subscribe to a message at initialization.

The first s field contains the name of the pipe. The second s field contains the macro name of the message. The 32 bit hexadecimal number is the error code returned by CFE.

MPU9250_MSGLEN_ERR_EID 

'Rcvd invalid msgLen: msgId=0x%08X, cmdCode=%d, msgLen=%d, expectedLen=%d"

Event Message:
'Rcvd invalid msgLen: msgId=0x%08X, cmdCode=%d, msgLen=%d, expectedLen=%d"
Type: ERROR
Cause:

This event message is issued when the CFS MPU9250Task has received a message with an invalid length.

The first d field contains the message ID. The second d field contains the command code. The third d field contains the actual length. The fourth d field contains the expected length.
