Notes on Parameters to starting the cFE in Linux, OS X, and Cygwin

In the previous three sections, the command to start the cFE was shown as: 

...fsw/build/cpu1]$ ./exe/core-xxx.xxx PO 1

This section will explain the parameters to the cfe core (the "PO" and the "1").

The first parameter is the type of reset that you want the cFE to think it started with. It can either be a Power On reset (PO), or a Processor Reset (PR).

The second parameter is the type of reset subtype you want the cFE to start with:

1 = Power on
2 = Push Button Reset
3 = Hardware Special Command Reset
4 = Watchdog Reset
5 = Reset Command

These are useful for desktop development. With these parameters, one can tell the cFE to try and recover CDS and reserved memory area (PR), or clear it out (PO). Since getting the reset type and subtype is available to cFE applications through an API, functionality of the application can be tested on the desktop.
