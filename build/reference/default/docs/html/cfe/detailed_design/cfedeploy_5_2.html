Populating the Local Directory

rdl Directory

  1. Extract all the files from MKS into the DESTINATION 'rdl/template' directory.
  2. Extract the '.rdl' files listed below from MKS 'rdl' into DESTINATION 'rdl' directory.
    • ccsds_header.rdl
    • cfe_user_tlm.rdl
    • cfe_shell_tlm.rdl
    • ccsds_header_no_append.rdl
    • pseudo_tlm_file_info_3839.rdl
    • cfe_cmd_types.rdl
    • cfe_file_header.rdl
    • user_cmd.rdl
    • cfe_tbl_header.rdl
    • user_defs.rdl
    • cfe_types.rdl
    • user_tlm.rdl
    • cfe_user_cmd.rdl
  3. Edit the user_cmd.rdl and user_tlm.rdl files that exist in the rdl directory. Make sure that they include the cfe_user_cmd.rdl and cfe_user_tlm.rdl files. These files specify what command and telemetry rdls to include in the ASIST databases.
  4. Create a link to the following files located in the appropriate flight software directory(ies). The directories listed below are the locations in the cFE MKS repository. The actual locations may vary depending upon the mission or OS. 
            -> ln -s .../fsw/Build/inc/app_mission_cfg.h .
        -> ln -s .../fsw/Build/inc/cfe_mission_cfg.h .
        -> ln -s .../fsw/Build/cpu3/cfe/config/inc/cfe_platform_cfg.h .
        -> ln -s .../fsw/Build/cpu3/cfe/config/inc/osconfig.h .

prc Directory

  1. Extract the 'template*.prc' files from MKS into the DESTINATION 'prc/template' directory.
  2. Extract the 'local/prc' files from MKS into the DESTINATION 'prc' directory.
  3. Create a link to the following files located in the appropriate flight software directory(ies). The directories listed below are the locations in the cFE MKS repository. The actual locations may vary depending upon the mission or OS. 
            ln -s .../fsw/Build/inc/cfe_mission_cfg.h .
        ln -s .../fsw/Build/cpu3/cfe/config/inc/cfe_platform_cfg.h .
        ln -s .../fsw/Build/cpu3/cfe/config/inc /osconfig.h .
  4. Change the file permissions on all procs so that they are writable. (chmod 666 *.prc)
  5. Create links to the appropriate flight software *events.h files that are included in the most test procedure files. 
            ln -s .../fsw/cfe-core/src/inc/*events.h .
        ln -s .../fsw/cfe-apps/to/to_events.h .
        ln -s .../fsw/cfe-apps/tst_es/tst_es_events.h .
        ln -s .../fsw/cfe-apps/tst_evs/tst_evs_events.h .
        ln -s .../fsw/cfe-apps/tst_sb/tst_sb_events.h .
        ln -s .../fsw/cfe-apps/tst_tbl/tst_tbl_events.h .
        ln -s .../fsw/cfe-apps/tst_tbl2/tst_tbl2_events.h .
        ln -s .../fsw/cfe-apps/tst_time/tst_time_events.h .

Other Local Directories

  1. Extract the local 'data', 'image', and 'tools' files (if any) from MKS into the DESTINATION 'data', 'image', and 'tools' directories.
  2. Change the file permissions to the appropriate level for any files extracted above.

  3. Enter the following commands to create other required directories: 

            mkdir test_logs
        mkdir apps
        cd apps
        mkdir cpu1
        mkdir cpu2
        mkdir cpu3

  4. Copy the test application object files from cfelnx to the apps/cpux directories. NOTE: This step requires a developer account on cfelnx with the privileges to create the test application object files required to execute test applications.
