288 /*
289  * Boolean type for compatibility --
290  *
291  * Note it is a bad idea to typedef "bool" or "boolean" -- MANY other projects
292  * and libraries also define a boolean type due to the lack of a standard bool in C89.
293  * But calling it simply "bool" or "boolean" almost guarantees a namespace conflict
294  * if trying to use OSAL with one of those other existing projects.
295  *
296  * RTEMS 4.11 no longer defines boolean type by default (deprecated) probably also
297  * due to the high likelihood of name conflicts.
298  *
299  * In order to preserve compatibility for apps written against prior versions of
300  * OSAL, the name "boolean" is typedefed as well, but this may be turned off
301  * in a future version whenever appropriate.
302  */
